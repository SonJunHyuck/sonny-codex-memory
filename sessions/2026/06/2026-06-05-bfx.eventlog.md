# 2026-06-05 bfx.eventlog

## 의도

BFX EventLog 작업을 여러 디바이스와 프로젝트 채팅에서 이어갈 수 있도록 Git 기반 Codex 메모리 저장소를 어떻게 구성할지 정한다.

## 변경

- `sonny-codex-memory`를 장기 메모리와 인수인계 저장소로 정했다.
- 실제 BFX 구현은 `ProjectBFX`에서 진행해야 한다고 확인했다.
- 메모리 컨텍스트를 project/topic directory layout으로 정리하기로 했다.
- 특히 미래 Notion 문서화를 위해 세션을 가벼운 원천 기록으로 유지하기로 했다.
- 작업 기억과 독자를 위한 문서 사이의 다리로 topic-level Notion 재료를 추가했다.
- 저장소 identity는 안정적으로 유지하고, 로컬 체크아웃 경로는 디바이스별 힌트로 다루도록 경로 모델을 수정했다.

## 결정

- 메모리 저장소와 프로젝트 저장소는 서로 다른 역할을 가진다.
- BFX EventLog 컨텍스트는 `contexts/bfx/eventlog/` 아래에 둔다.
- `current.md`, `decisions.md`, `timeline.md`를 주요 이어가기 파일로 사용한다.
- `sessions/`는 나중에 시스템이 왜 만들어졌고 결정이 어떻게 생겼는지 복기할 때 유용하다.
- `notion.md`는 구조화된 문서화 재료를 모은다.
- 작업이 PC와 Mac을 오갈 것이므로 절대 로컬 경로를 기준 경로로 취급하지 않는다.

## Notion 재료

- 메모리 저장소를 구현 저장소가 아니라 "인수인계 계층"으로 설명한다.
- 시작과 종료 운영 흐름을 문서화한다.
  - 시작: 메모리 저장소 pull, 컨텍스트 읽기, 프로젝트 저장소 열기.
  - 종료: 컨텍스트 업데이트, 메모리 저장소 commit/push.
- raw chat log를 기본적으로 저장하지 않는 이유를 문서화한다.
