# 2026-06-05 bfx.eventlog

## Intent

Define how a Git-backed Codex memory repo should be organized so BFX EventLog work can continue across devices and project chats.

## Changed

- Established `sonny-codex-memory` as the long-term memory and handoff repo.
- Confirmed actual BFX implementation should happen in `ProjectBFX`.
- Chose a project/topic directory layout for memory contexts.
- Decided to keep sessions as lightweight source notes, especially for future Notion documentation.
- Added topic-level Notion seeds as a bridge from working memory to reader-facing documentation.
- Corrected the path model so repo identity is stable and local checkout paths are device-specific hints.

## Decisions

- Memory repo and project repo have separate roles.
- BFX EventLog context lives under `contexts/bfx/eventlog/`.
- `current.md`, `decisions.md`, and `timeline.md` are the main continuity files.
- `sessions/` is useful when later reconstructing why a system was built and how decisions emerged.
- `notion.md` should collect structured documentation seeds.
- Absolute local paths should not be treated as canonical because work will move between PC and Mac.

## Notion Seeds

- Explain the memory repo as an "handoff layer" rather than an implementation repo.
- Document the start and end workflow:
  - Start: pull memory repo, read context, open project repo.
  - End: update context, commit, and push memory repo.
- Document why raw chat logs are not stored by default.
